## Module 7 — Daily software engineering habits in VS Code

**Rendered Markdown:** [Preview this step (full checklist)](command:engineeringTutorial.previewWalkthroughDailyPractices)

Connect the syllabus to **how you work every day**.

### Source control

- **Source Control** view: small, reviewable commits; write messages that explain *why*.
- **Diff editor**: read every hunk before push—same discipline as reviewing a teammate’s PR.

### Navigation & search

- **Go to Definition / References** (`F12` / `⇧F12`): treat the codebase as a graph, not a folder tree.
- **Search across files** (`Ctrl+Shift+F`): find call sites and patterns before you refactor.

### Quality loop

- **Problems** + **Format Document** + **Save**: keep feedback tight; automate what repeats (see `tasks.json` in your projects).

### Command Palette

- `⇧⌘P` / `Ctrl+Shift+P`: if you do something twice, learn the command name or bind a key.

### Teaching checklist

| Habit | VS Code surface |
|--------|------------------|
| Traceability | SCM, Timeline |
| Correctness | Problems, Testing extensions |
| Speed | Multi-cursor, rename symbol |
| Collaboration | PR extensions, Live Share (if used) |
