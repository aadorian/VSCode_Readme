## Module 2 — Collaboration workflows

Distributed teams align through **issues**, **pull requests**, and **templates** so reviewers get consistent context.

### Read in the repo

Open the **pull request template** used in the upstream-style workflow:

- [pull_request_template.md](command:engineeringTutorial.openPullRequestTemplate)

Explore on disk (Explorer sidebar):

- `education/collaboration-workflows/ISSUE_TEMPLATE/` — bug vs feature structure.
- `education/collaboration-workflows/pr-template-fixtures-from-github-extension/` — how the **GitHub extension** discovers PR templates.

### Practice in VS Code

1. Open **Source Control** (`Ctrl+Shift+G` / `⇧⌘G`).
2. Install **GitHub Pull Requests and Issues** if you use GitHub; compare its flow to the docs you just read.

### Software engineering takeaway

**Templates encode process** — they are executable documentation for humans and tools.
